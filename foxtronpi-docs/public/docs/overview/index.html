    1. 總則說明#

    1.1 目的#

    為突破目前汽車產業軟體從業人員不足與封閉的開發環境，「FoxtronPi」平台以靈活強大的軟體支援，提供車用軟體開發的應用平台。「FoxtronPi」採用 Diagnostic communication over Internet Protocol 技術 (後稱 DoIP)，進行車輛平台動態控制，平台具備明確的功能定義規範及不同階段的可控參數，可協助車用軟體開發者在平台上快速布置演算邏輯，實車實現目標算法。本文各章節將依序介紹『總則說明』、『系統架構』、『技術規格』及『名詞解釋』。

    1.2 安全需知#

    FoxtronPi 為提供外部控制器操作車輛的平台，由於此操作涉及車輛動態，為確保人車安全，請務必遵循下列規範：

    1. 測試場域：僅能在空曠且封閉之場域進行測試，車輛行駛時速不得高於 30km/h，禁止於公共道路或人車混行區域操作。

    2. 法規遵循：不得有任何違法或超速行為，並遵守所在地相關法規與場地管理規定。

    3. 文件熟悉：操作前請完整閱讀並熟悉本文件之流程與限制條件，未依流程進行，系統可能拒絕控制或產生風險。

    4. 初次調參建議： 初次使用或參數變更請由低速開始 (建議5-10km)，逐步提升；每次改動僅變更單一參數並觀察回饋。

    5. 人員配置：切勿單人、無人作業，需採雙人作業 (安全駕駛＋測試員)，安全駕駛須具備相應駕照與經驗，全程握持方向盤、腳不離煞車，可隨時接管。

    6. 緊急停止：車輛配置紅色緊急按鈕，可執行緊急停止機制，按下紅色緊急按鈕可即時生效，請於每次測試前演練，詳細說明請參閱 9. 紅色緊急按鈕操作說明。

    7. 車況檢查：測試前完成制動、轉向、輪胎、電量與 DTC 狀態檢查；若存在異常，不得進行動態測試。

    8. 環境條件：避免於雨雪、低附著或視線不良環境操作；確保撤離通道暢通、場域設有明確邊界與緩衝區。

    9. 權限與保護：遵循寫入保護、解除流程，禁止未授權人員修改控制參數。

    10. 回復機制：每次測試結束，請復位控制，確認車輛回到安全靜止狀態後方可離場。
